1. Etiqueta: <article>

Caso de Uso: Contenido independiente (post de blog, comentario). Debe tener sentido si se extrae del contexto del sitio.

Código de Ejemplo

<article class="post">
    <h2>Nuevas Tendencias en Diseño Web</h2>
    <p>El uso de Flexbox y Grid sigue en aumento...</p>
    <footer>Publicado el 2025-01-01</footer>
</article>
Resultado en el Navegador
Documentación W3C

Especificación <article> (W3C)

2. Etiqueta: <section>

Caso de Uso: Agrupa contenido relacionado lógicamente dentro de un documento. Debe tener un encabezado propio.

Código de Ejemplo

<section id="sobre-nosotros">
    <h2>Sobre Nosotros</h2>
    <p>Somos una iniciativa dedicada a la educación web.</p>
</section>
Resultado en el Navegador
Documentación W3C

Especificación <section> (W3C)

3. Etiquetas: <figure> y <figcaption>

Caso de Uso: Se utiliza para marcar contenido que es referenciado desde el cuerpo principal, como fotos, diagramas o código, y su leyenda o título (`figcaption`).

Código de Ejemplo

<figure>
    <img src="mi-diagrama.png" alt="Diagrama de flujo">
    <figcaption>Figura 1: Diagrama de flujo del sitio web.</figcaption>
</figure>
Resultado en el Navegador (en el logo de la imagen estaria la imagen deseada)
Documentación W3C

Especificación <figure> (W3C)

4. Etiqueta: <input type="date">

Caso de Uso: Crea un campo de formulario que permite al usuario seleccionar una fecha con un selector visual nativo del navegador (calendario).

Código de Ejemplo

<label for="fecha-reserva">Selecciona una fecha:</label>
<input type="date" id="fecha-reserva" name="reserva">
Resultado en el Navegador
Documentación W3C

Especificación <input type="date"> (W3C)

5. Etiquetas: <details> y <summary>

Caso de Uso: Permite crear un *widget* de divulgación interactivo donde el usuario puede abrir y cerrar contenido adicional (útil para Preguntas Frecuentes).

Código de Ejemplo

<details>
    <summary>Ver más sobre la actividad</summary>
    <p>Esta actividad es obligatoria para el 1er trimestre.</p>
</details>
Resultado en el Navegador
Documentación W3C

Especificación <details> y <summary> (W3C)

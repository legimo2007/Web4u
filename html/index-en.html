1. Tag: <article>

Use Case: Independent, self-contained content (blog post, comment). Should make sense if extracted from the site context.

Example Code

<article class="post">
    <h2>New Trends in Web Design</h2>
    <p>The use of Flexbox and Grid continues to increase...</p>
    <footer>Published on 2025-01-01</footer>
</article>
Browser Output
W3C Documentation

<article> Specification (W3C)

2. Tag: <section>

Use Case: Groups logically related content within a document. It should generally have its own heading.

Example Code

<section id="about-us">
    <h2>About Us</h2>
    <p>We are an initiative dedicated to web education.</p>
</section>
Browser Output
W3C Documentation

<section> Specification (W3C)

3. Tags: <figure> and <figcaption>

Use Case: Used to mark content referenced from the main body, such as photos, diagrams, or code, and its caption or title (`figcaption`).

Example Code

<figure>
    <img src="my-diagram.png" alt="Flowchart Diagram">
    <figcaption>Figure 1: Website Flowchart.</figcaption>
</figure>
Browser Output (The desired image would be in the image logo)
W3C Documentation

<figure> Specification (W3C)

4. Tag: <input type="date">

Use Case: Creates a form field that allows the user to select a date using a native browser visual selector (calendar).

Example Code

<label for="reservation-date">Select a date:</label>
<input type="date" id="reservation-date" name="reservation">
Browser Output
W3C Documentation

<input type="date"> Specification (W3C)

5. Tags: <details> and <summary>

Use Case: Allows creating an interactive disclosure widget where the user can open and close additional content (useful for FAQs).

Example Code

<details>
    <summary>See more about the activity</summary>
    <p>This activity is mandatory for the 1st trimester.</p>
</details>
Browser Output
